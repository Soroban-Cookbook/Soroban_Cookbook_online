# What is Soroban?

<div className="badge-beginner">Beginner</div>
<div className="text-sm text-gray-600 dark:text-gray-400 mt-2">‚è±Ô∏è 5 min read</div>

Soroban is Stellar's smart contracts platform that enables developers to build decentralized applications (dApps) using the **Rust programming language**. Built on Stellar's proven blockchain infrastructure, Soroban combines the power of Rust with Stellar's speed and low transaction costs.

## Why Soroban?

### üöÄ Performance
- **Sub-second finality**: Transactions confirm in 3-5 seconds
- **Low costs**: Fractions of a cent per transaction
- **Scalability**: Handles thousands of transactions per second

### üîí Security
- **Memory safety**: Rust's ownership system prevents common bugs
- **Type safety**: Catch errors at compile time, not runtime
- **Battle-tested**: Built on Stellar's infrastructure (launched 2014)

### üõ†Ô∏è Developer Experience
- **Familiar tooling**: Uses standard Rust tools (cargo, rustc)
- **Rich ecosystem**: Access to Rust's extensive library ecosystem
- **Clear APIs**: Well-documented SDK with intuitive abstractions

## How is Soroban Different?

### vs. Solidity (Ethereum)

| Feature | Soroban (Rust) | Solidity |
|---------|----------------|----------|
| **Language** | Rust | Custom (Solidity) |
| **Memory Safety** | Compile-time guarantees | Runtime checks |
| **Gas Model** | Predictable, low-cost | Variable, can spike |
| **Finality** | 3-5 seconds | 12+ minutes |
| **Ecosystem** | Rust crates | Solidity-specific |

### vs. Traditional Web2

Soroban brings decentralization benefits while maintaining developer familiarity:
- Write code like normal Rust (no blockchain-specific syntax)
- Test with standard `cargo test`
- Deploy to global, censorship-resistant network

## Core Concepts at a Glance

Don't worry if these don't make sense yet‚Äîwe'll cover each in detail:

1. **Contracts**: Self-contained Rust modules compiled to WebAssembly
2. **Storage**: Persistent, temporary, and instance data on-chain
3. **Authorization**: Cryptographic signatures for access control
4. **Events**: Emit logs for off-chain indexing and monitoring
5. **Invocations**: Call other contracts or Stellar built-ins
